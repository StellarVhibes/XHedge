# Smart Contract Test Snapshots

## Overview
When running `cargo test` in this smart contract repository, the testing framework generates test snapshots (typically JSON files) that record the execution state, events, and invariants of the contract.

## Ignore Rules
The `test_snapshots/` directory is automatically ignored by git via `.gitignore`. 
**Contributors must NOT commit test snapshot files.**

## Why?
- **Noise:** Snapshots contain large amounts of autogenerated output that clutter pull requests.
- **Merge Conflicts:** They frequently cause severe merge conflicts when multiple contributors run tests and alter the exact execution paths or gas fees.
- **Local Verification:** Snapshots are strictly for local verification, debugging, and framework use.

Please ensure your local `.gitignore` is active and you do not use `git add -f` to force add any files from the `test_snapshots/` directory.
